# 课程资料页 PDF 直读式展示设计

## 背景

目标页面是 `/learn/course/2?tab=materials`。用户希望“学习资料”页签像古月居书站一样，把文件本身作为主阅读对象，而不是继续以资料卡片、统计组件和弹窗预览为中心。

此前已有一版“三栏资料工作台”方向，但用户反馈组件过多、文件展示累赘、阅读体验缺少连贯感。因此本设计把方向收敛为“单文件沉浸阅读”：左侧资料目录，中间当前文件原文，右侧当前文件导读。

## 设计目标

1. PDF 直接展示在页面中间，作为默认主角。
2. 左侧目录只承担“切换资料”的功能，不再展示过多摘要和统计。
3. 右侧只展示当前文件相关信息，不做完整资源中心。
4. 保留课程平台语境：资料仍按阶段组织，PDF、视频、链接都能进入同一个阅读框架。
5. 正式实现前不改后端接口、不改数据库、不改教师端上传流程。

## 核心结构

### 左侧：资料目录

- 展示课程阶段，例如“阶段 1、阶段 2、阶段 3”。
- 每个阶段下列出资料标题、类型和一条极短元信息。
- 点击资料后，切换中间阅读器的当前文件。
- 不再放资料卡片、统计卡、模式切换和大段说明。

### 中间：当前文件阅读器

- 顶部是紧凑文件栏：阶段路径、文件标题、类型、页数或时长、打开原文件、下载。
- PDF 资料使用 `<object>` 或 `<iframe>` 直接嵌入，文件地址复用 `getPublicMaterialFileUrl(material.id)`。
- 如果浏览器无法内嵌 PDF，显示中文降级提示，并提供“新窗口打开”和“下载”。
- 视频资料进入同一位置的站内播放器。
- 链接资料进入简化的链接落地视图，提供摘要和打开入口。

### 右侧：当前文件导读

- 当前文件摘要。
- 文件类型、页数或时长。
- 本页目录，第一阶段先用资料摘要或人工配置内容，不做自动 PDF 目录解析。
- 阅读重点，优先复用资料摘要；后续如需要，可在后台新增导读字段。
- 相关资料最多 2 到 3 个，只作为辅助跳转。

## 需要砍掉的内容

1. 顶部阅读模式切换。
2. 大面积统计卡片。
3. 中间资料卡片流。
4. 右侧完整资源中心。
5. 同时展示多个文件详情。

这些内容会把页面重新拉回“资料库”形态，不利于学生连续阅读 PDF。

## 正式实现建议

### 涉及文件

- `frontend/src/views/CourseDetailView.vue`
- 可选新增 `frontend/src/components/learn/MaterialInlineReader.vue`
- 可选新增 `frontend/src/components/learn/MaterialGuideRail.vue`
- `docs/superpowers/project-map.md`
- `frontend/tests/public-learning-static.test.mjs`

如果只做第一阶段，可以先在 `CourseDetailView.vue` 内完成，不急着拆组件。只有当阅读器逻辑开始同时处理 PDF、视频、链接和降级状态时，再拆出 `MaterialInlineReader.vue`。

### 数据来源

- 课程阶段资料：继续使用 `course.value?.stages`。
- 未分类资料：继续使用 `course.value?.uncategorized_materials`。
- 资料完整列表：继续使用 `materials`。
- PDF 文件地址：公开学习页继续使用 `getPublicMaterialFileUrl(material.id)`。
- 登录用户或教师端如果有不同文件 URL，沿用当前页面已有 `materialFileUrl(material)` 分支。

### 默认选中逻辑

1. 优先选中第一份 PDF。
2. 如果没有 PDF，选中第一份视频。
3. 如果只有链接，选中第一份链接。
4. 如果没有资料，显示简洁空状态。

### 移动端

- 不使用三栏。
- 顺序为：当前文件栏、资料目录横向列表、文件阅读器、导读信息。
- PDF 阅读器保持最大可用高度，避免目录和导读挤占阅读空间。

## 验收标准

1. 打开 `/learn/course/2?tab=materials` 后，中间直接显示当前 PDF 文件。
2. 左侧目录点击资料后，中间阅读器同步切换。
3. 当前资料是 PDF 时，不再必须点击弹窗才能看正文。
4. 右侧只围绕当前文件，不展示庞杂资源列表。
5. 页面没有资料卡片瀑布流的主视觉。
6. 移动端内容不互相遮挡，PDF 阅读区仍然是主区域。
7. 不改变后端接口、数据库、上传流程。

## 服务器部署影响

当前只新增设计文档和讨论原型，不影响服务器部署，不需要拉代码、重启服务、重新构建前端、执行数据库迁移、调整环境变量或修改 Nginx 配置。

如果后续按本设计改正式前端页面，则服务器需要拉取前端代码并重新构建、部署静态资源；不需要后端重启和数据库迁移。
