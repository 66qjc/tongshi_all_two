# 代码审查问题全量修复设计

## 一、背景

2026-07-10 对当前未提交工作区进行了后端安全与数据一致性、前端流程与接口契约、缓存与数据库兼容、测试与部署四个角度的审查。审查确认当前新增功能中同时存在权限越界、完成状态误判、共享数据被级联删除、前后端鉴权契约错位、并发写入丢失、缓存实际失效和旧库索引缺失等问题。

本设计不扩展新的业务功能，目标是把已经落地但行为不完整的功能修正到可部署状态。实施按风险拆分，避免在同一批修改中同时改动过多共享边界。

## 二、目标

1. 修复作业完成状态、课程共享题库和课时进度的业务正确性。
2. 修复教师通知、密码重置和软删除用户的权限与登录安全边界。
3. 修复随机点名、通知中心、作品入口和文件预览的前后端契约。
4. 使私有大文件预览恢复短时鉴权与 Range 流式读取能力。
5. 使 Redis 缓存具备稳定键、可序列化数据、故障降级和可验证失效语义。
6. 为既有 MySQL 数据库补齐新增索引，并避免多进程同时执行兼容 DDL。
7. 恢复全部受影响的自动化测试，并补充能复现本轮缺陷的回归测试。

## 三、范围

### 3.1 范围内

- 作业答题、自动计分和 `TaskCompletion` 完成语义。
- JWT `token_version`、所有密码重置路径、软删除用户登录校验。
- 教师发送通知时的学生归属校验和站内跳转约束。
- 课程软删除、七类核心资源回收站、共享题库转挂和恢复语义。
- 课时完成、访问次数、并发心跳和教师学习分析。
- 随机点名、作品入口、通知跳转、通知刷新和异常加载状态。
- 公开与私有 PDF/视频的鉴权和流式预览。
- Redis 基础缓存、故障降级、缓存失效和部署开关。
- 忘记密码失败计数在 Redis 开启时跨 worker 共享。
- `schema_compat.py` 对既有 MySQL 的索引补齐和启动锁。

### 3.2 范围外

- 不实施完整人工批改、作业附件、成绩申诉等尚未落地的评分系统二期能力。
- 不新增后台定时任务进程；作业提醒和过期通知继续由外部调度调用现有服务函数。
- 不重做页面视觉风格，不更换 Vue 状态管理或 Element Plus 组件体系。
- 不切换 S3、MinIO 或其他文件存储后端。
- 不引入 Alembic；本轮继续沿用项目现有 `schema_compat.py`，但补齐其幂等性和并发保护。

## 四、方案选择

### 4.1 采用：按风险分阶段修复

先修权限和数据正确性，再修用户流程和文件预览，最后修缓存与数据库兼容。每个阶段必须有独立失败用例、最小实现和阶段回归，上一阶段通过后才进入下一阶段。

选择原因：当前工作区已有大量未提交修改，按风险拆分能降低交叉回归，也便于定位某一修复引入的新问题。

### 4.2 不采用：仅修现有失败测试

现有测试没有覆盖教师跨班通知、旧 JWT、公开资料预览、Redis ORM 序列化和既有 MySQL 索引，因此只追求现有测试变绿会保留真实缺陷。

### 4.3 不采用：一次性重构全部相关服务

一次性重构可获得更整齐的抽象，但会扩大当前脏工作区的冲突面，不符合项目“只改当前任务直接需要内容”的约束。

## 五、目标行为设计

### 5.1 作业完成与自动评分

- `QuizAttempt` 继续记录每次答题事实。
- 自动评分可以在每次答题后重新计算，但不得因为存在部分答题记录就创建 `TaskCompletion`。
- 只有任务题目集合全部存在至少一条答题记录时，才允许创建 `TaskCompletion`。
- `mark_completed()` 即使发现已有完成记录，也必须先校验任务时间窗口和全部题目完成情况，避免历史脏数据绕过校验。
- 单次答题写入、分数计算和完成记录更新必须位于同一数据库事务；内部辅助函数只 `flush`，最外层业务服务沿用项目现有风格只提交一次，异常时统一回滚。
- 教师完成报告仍以 `TaskCompletion` 表示正式完成，未完成学生的临时得分从 `QuizAttempt` 聚合，不混入完成集合。

### 5.2 密码与登录安全

- 登录和 `get_current_user` 必须过滤 `User.deleted_at IS NULL`。
- 登录态修改密码、密保答案重置、教师/管理员审批重置、管理员重置教师密码都必须递增 `token_version`。
- 登录态修改密码成功后，后端使用新版本签发一个新 JWT，并在现有成功响应的 `data` 中附加 `access_token`；前端立即替换本地 Token，使当前设备可以继续完成密保设置。
- 后端 Token 轮换与前端 Token 接收必须在同一阶段上线；`ChangePasswordView`、首次登录弹窗和个人中心等所有改密入口统一通过 auth store 保存新 Token。
- 匿名重置和管理员代重置不向被重置用户签发 Token，用户必须使用新密码重新登录。
- 密码变更与审计日志在同一事务提交；审计失败不得留下“密码已改但无审计”的半完成状态。

### 5.3 教师通知权限

- 管理员可以向任意未删除学生发送通知。
- 教师只能向自己创建的班级中的未删除学生发送通知；批量接口逐个过滤并返回实际发送数和跳过数。
- 路由必须把 `current_user` 传给通知服务，不再使用匿名 `_` 依赖参数。
- `action_url` 仅允许空字符串或以单个 `/` 开头的站内路径，拒绝 `//`、协议 URL 和脚本方案。
- 作品审核通知统一跳转 `/create/project/{project_id}`。

### 5.4 共享题库与软删除

- 全站共享题目不能因教师课程删除而进入回收站。
- 删除课程前调用题目转挂逻辑，把该课程下未删除题目转挂到另一门未删除课程；无可用目标时拒绝删除并给出中文说明，避免静默丢题。
- 转挂只改变外键锚点，不改变题目贡献人；教师编辑权限依据 `Question.created_by`，不能转移给承接课程的所有者。
- 课程级联软删除只包含班级、资料和公告，不包含共享题目。
- `users`、`courses`、`classes`、`announcements`、`projects`、`materials`、`questions` 七类资源的正式删除入口统一调用软删除服务；管理员“彻底删除”仍执行物理删除。
- 班级、公告、作品、资料和用户软删除前不得物理清理关联表；恢复后关联记录必须完整。管理员删除被未删除作业引用的题目时必须拒绝，不能留下失效的 `question_ids`。
- 回收站路径参数统一接收字符串，服务层根据 ORM 主键类型安全转换；前端 `DeletedResourceItem.id` 使用 `string | number`。
- 恢复课程时只恢复同一删除批次的班级、资料和公告，不恢复已转挂的共享题目。
- 所有本轮涉及的查询补齐 `deleted_at IS NULL`，重点覆盖登录、通知目标、进度分析、课程/班级归属和教师管理入口。

### 5.5 课时进度与学习分析

- 首次写入使用数据库 upsert；既有记录使用原子累加，避免并发首次插入冲突和读改写丢失。
- `duration_seconds` 和 `view_count` 不在 Python 中基于旧值累加。
- `LessonProgressIn` 新增 `visit_started: bool = False`。前端每次真正进入一个课时只上报一次 `visit_started=true`，30 秒心跳与页面补报均为 `false`。
- `view_count` 只在 `visit_started=true` 时加一，教师端“浏览次数”保持真实访问语义。
- 视频课时继续按播放位置计算完成百分比。
- 不含视频的课时显示“完成本课”按钮，学生主动确认后上报 `progress_percent=100`；后端仍保留疑似快速完成标记，不把按钮隐藏作为防刷课手段。
- 页面隐藏与 `pagehide` 只允许一个最终补报通道；发送前原子取得本次尚未上报的时长，避免重复请求覆盖。
- 课程分析改为数据库聚合并对学生明细分页，避免加载 `学生数 × 课时数` 的全部 ORM 对象。
- 课程页组件实例不能因 `lesson_id` 查询参数变化而重建；一次真实课时切换只能产生一次 `visit_started=true`。

### 5.6 前端业务流程

- 随机点名先请求班级列表，选中班级后调用 `getClassStudents(classId)`；切换班级重新加载并清空本轮点名状态。
- `/create` 维持现有稳定契约：作品广场需要登录。移除该路由的 `public` 标记，与首页“登录后查看作品展示”文案一致。
- 通知中心把公告和系统通知加载封装为统一 `loadMessages()`，在 `finally` 中结束首屏加载；偏好只在首屏和保存后加载，轮询不得覆盖用户尚未保存的表单状态。
- 通知中心恢复 15 秒刷新、页面重新可见刷新和跨组件消息刷新事件；成功已读、全部已读和作业完成后广播刷新。
- 页头点击最近通知时先标记已读，再跳转；失败时保留未读状态并给出中文提示。

### 5.7 文件鉴权与流式预览

- 公开课程资料始终优先使用 `/api/public/learning/materials/{material_id}/file`，不能被响应中的 `file_id` 覆盖成私有接口。
- 私有文件新增 `POST /api/files/{file_id}/access-url`。该接口先使用正常 Bearer Token 完成文件级权限校验，再返回 5 分钟有效的文件专用签名 URL。
- 文件专用 Token 只包含 `sub`、`file_id`、`scope=file_access` 和短期 `exp`，不能用于其他 API，也不能访问不同文件。
- `/api/files/{file_id}` 只在当前文件路径接受文件专用查询 Token；普通长效 JWT 仍不得通过查询参数传递。
- PDF.js、`<video>`、`<object>` 和“打开原资料”统一使用签名 URL，浏览器可以继续发送 Range 请求，前端不再把受保护大文件完整转换为 Blob。
- URL 过期时前端重新申请一次；组件卸载时取消尚未完成的签名 URL 请求。
- 迁移范围覆盖所有 `/api/files/{file_id}` 消费者，包括课程资料、课时内嵌视频、作品图片与报告、教师审核、管理员 PDF 预览和资料原文件入口；通用 `resolveFileUrl()` 不再追加普通登录 JWT 查询参数。
- 文件权限查询必须过滤已软删除的课程、资料、作品、预览和用户；资源删除后旧签名 URL 也不能继续读取文件。

### 5.8 Redis 缓存

- 新增 `REDIS_ENABLED`，默认 `false`；未启用时完全跳过 Redis 调用，不产生连接告警和额外延迟。
- 缓存键通过显式 `key_builder` 生成，禁止包含 `Session`、对象地址或未展开占位符。
- 只缓存已经格式化的 JSON 字典/列表，不缓存 ORM 实例或 ORM 元组。
- Redis 连接失败后进入短暂失败冷却，冷却期直接访问数据库，避免每个请求重复等待两次超时。
- 模式失效使用增量 `SCAN`，禁止生产路径使用 `KEYS`。
- 第一批只覆盖公共课程列表、课程详情和班级列表；每个缓存场景必须有命中、更新失效、删除失效和 Redis 不可用降级测试。
- 忘记密码失败次数在 Redis 开启时使用带 TTL 的原子计数，成功后清除；Redis 关闭时仅保留单 worker 内存降级，多 worker 部署必须启用 Redis。

### 5.9 数据库兼容

- 新增 `_ensure_index`，为既有 MySQL 和 SQLite 表幂等补齐本轮 ORM 声明的关键索引。
- 至少补齐 `quiz_attempts(user_id, announcement_id)`、`quiz_attempts(user_id, question_id)`、`lesson_progress(user_id, course_id)`、软删除过滤字段、`questions.stem_hash`、通知筛选字段和审计筛选字段索引。
- 对既有 `questions` 数据重算规范化题干 SHA-256；不能只回填空值而保留旧 32 位 MD5。编辑、管理员创建和批量导入均复用同一哈希函数。
- `stem_hash` 保持普通索引而不是唯一约束，业务查重继续允许在明确规则下处理“同题干但答案不同”的候选关系。
- MySQL 在 `Base.metadata.create_all()` 之前获取命名数据库锁，并在同一连接中执行建表、兼容 DDL、哈希升级和索引补齐；完成或异常时释放。SQLite 测试环境跳过命名锁。

## 六、错误处理

- 权限越界统一使用现有 `BusinessException(403, "无权执行此操作")` 风格，并按具体资源保留项目已有中文文案。
- 资源不存在或已删除继续返回 404，避免泄露跨教师资源是否存在。
- 作业未答完返回 400，并使用现有“请先完成全部题目后再标记完成”文案。
- Redis、签名 URL刷新和通知刷新失败不得阻断核心数据库读取；前端必须显示用户可理解的中文提示。
- 数据库唯一约束或并发写入异常必须回滚，并转换为稳定业务响应，不向前端暴露 SQL 错误。

## 八、测试与验收

### 8.1 后端自动化

- 多题作业只答一题时不能完成；全部答完后只创建一条完成记录。
- 教师不能通知其他教师班级学生，管理员可以。
- 四种密码变更路径均使旧 JWT 失效；登录态改密返回的新 JWT 可继续使用。
- 软删除用户无法登录或使用旧 JWT。
- 删除教师课程不减少全站共享题目数，恢复课程不复制题目。
- 两个并发课时首次上报只产生一条记录，累计时长和访问次数不丢失。
- 文件专用 Token 不能跨文件、不能调用普通 API，过期后返回 401。
- Redis 启用时命中和失效正确，禁用或故障时业务结果与直查一致。
- 两个 worker 使用同一 Redis 时共享忘记密码失败次数，关闭 Redis 时明确仅支持单 worker 语义。
- 既有旧表运行兼容层后具备目标索引和题干哈希。

### 8.2 前端自动化

- 恢复当前全部静态测试，包括消息刷新、课程布局、品牌文案和术语一致性。
- 新增随机点名接口契约、改密 Token 轮换、非视频课时完成、公开资料 URL 优先级和签名 URL 使用测试。
- `npm run type-check` 和 `npm run build` 必须通过。

### 8.3 浏览器验收

- 学生、教师、管理员三类账号完成登录与改密流程。
- 教师随机点名能加载班级学生并完成一轮不重复点名。
- 学生完成文本课时和视频课时，教师统计中的完成率、时长、访问次数正确。
- 游客打开公开 PDF，学生打开私有大 PDF/视频；确认首屏可见、Range 请求生效、切换资料不继续下载旧文件。
- 通知中心刷新、已读和作品详情跳转正确。
- 管理员恢复字符串 ID 用户和课程，危险操作确认与失败提示正常。

### 8.4 数据库与性能

- 使用旧结构 SQLite 夹具验证兼容升级；在 MySQL 测试库核对索引、哈希回填和迁移锁。
- Redis 开启与关闭各跑一次受影响接口测试。
- 课程分析至少使用 100 名学生、20 个课时的数据检查查询数量、响应大小和耗时。

## 十、服务器部署影响

正式上线需要：

1. 备份 MySQL 数据库和上传文件目录。
2. 服务器拉取代码并安装后端依赖。
3. 根据部署选择配置 `REDIS_ENABLED`；启用时配置 Redis 地址、密码和超时。
4. 单实例先执行兼容升级，确认题干哈希回填和索引创建完成，再启动多 worker。
5. 重启后端服务。
6. 重新构建并部署前端静态资源。
7. 验证签名文件 URL、Range 请求、登录改密、课程删除、作业完成和通知权限。

本设计不要求调整 Nginx 路由；现有内部文件传输配置继续保留。若服务器当前未部署 Redis，可保持 `REDIS_ENABLED=false`，核心功能必须正常运行。

## 十一、风险与回滚

- 共享题库转挂、题干哈希回填和软删除属于数据修改，部署前必须备份数据库。
- 文件签名 URL 涉及鉴权边界，必须先完成跨文件和过期 Token 测试再上线。
- 旧库索引创建可能锁表，应在低峰期执行并监控 MySQL。
- 每个阶段独立提交和部署；某阶段验证失败时只回滚该阶段代码。数据库回滚优先恢复备份，不使用删除列或删除索引作为自动回滚手段。
