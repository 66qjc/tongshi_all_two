# 教师端前端验收要点

## 导航与命名

教师端导航固定为 10 项：概述、课程管理、班级管理、发布题目、学生成绩、任务完成、作品审核、资料管理、学生管理、题库管理。

- 不再出现“任务发布”“学生数据”作为教师端入口名。
- 不再出现章节相关入口、筛选项或详情页。

## 页面规则

- 课程管理：展示课程名称、资料数、题目数、创建时间；提供查看资料、编辑、删除。
- 班级管理：班级按课程创建和筛选；查看学生只读；学生增删进入学生管理页。
- 发布题目：只发布题目，不提供公告正文；目标班级支持多选且来自同一课程。
- 学生成绩：承接 `/teacher/grades?tab=tasks&task_id=...`，直接展示任务完成情况。
- 任务完成：从发布题目页跳转，展示指定任务的完成/未完成学生列表及超时情况。
- 资料管理：上传和筛选按课程，不再按章节；教师可删除自己课程下的资料，公共课程同步到副本里的资料也可删除，但不会影响公共源内容。
- 资料管理和教师课程详情中的资料必须展示图文信息，包括封面、摘要、类型、大小、页数或时长、预览状态；不能只展示标题。
- 教师端资料预览与学生端共用站内预览组件，教师可检查 PDF / 视频实际打开效果。
- 题库管理：题目按课程和题型筛选，不再按章节；教师可新增和编辑自己课程题目，不提供删除入口，删除只由管理员端处理。

## 交互底线

- 危险操作必须确认。
- 后端拒绝删除课程或班级时，页面必须给出可理解提示。
- 空列表需要有中文空状态。
- 所有页面文案使用正常中文，禁止乱码和英文占位文案。
