# 前端与教师端验收准则

本文件承接 `AGENTS.md` 中的前端细则，重点约束教师端后台管理页面的 UI、文案和验收路径。

## 教师端 UI 原则

- 保持后台管理系统风格，清晰、稳定、易扫描。
- 不做营销页风格，不使用大面积装饰性视觉。
- 表格列名要短而清楚。
- 主按钮使用明确动作，例如“新增班级”“上传资料”“新增题目”“发布任务”。
- 删除、移除、驳回等危险操作必须有确认弹窗。
- 空状态必须说明当前没有什么，以及下一步可以做什么。
- 加载失败、保存失败、导入失败必须有用户可理解的提示。
- 筛选项、表头、分页、弹窗标题和表单标签应尽量与现有教师端页面风格一致。
- 页面文案、按钮、提示、弹窗、表格列名必须使用正常中文，禁止乱码。

## 前端实现要求

- 统一使用 Vue 3 `<script setup lang="ts">`。
- 优先复用已有 API 封装、页面结构和 Element Plus 交互模式。
- 不引入新的 UI 框架。
- 不为了修文案而大规模重写业务逻辑。
- 不修改与当前任务无关的学生端功能。
- 不静默吞掉错误，至少要给出用户可理解提示。

## 推荐验收路径

前端改动优先检查这些页面：

- `/teacher/dashboard`
- `/teacher/materials`
- `/teacher/questions`
- `/teacher/courses`
- `/teacher/students`
- `/teacher/classes`
- `/teacher/announcements`
- `/teacher/reviews`
- `/admin/teachers`
- `/admin/showcase`
- `/profile`
- `/change-password`

## 基础验收清单

- `frontend` 下执行 `npm run build` 是否通过。
- 修改过的教师端页面是否能正常打开。
- 页面没有乱码、残缺标签、明显错字。
- 表格、弹窗、按钮、Toast 提示文案正常。
- 空数据状态显示合理。
- 没有引入无关改动。

如果浏览器环境无法本地启动，必须在总结中说明哪些运行时路径没有验证。
