# 教师端前端验收要点

## 导航与命名

教师端导航固定为 10 项：概述、课程管理、班级管理、发布作业、学生成绩、作业完成、作品审核、资料管理、学生管理、题库管理。

- 不再出现“任务发布”“学生数据”作为教师端入口名。
- 不再出现章节相关入口、筛选项或详情页。
- 不再出现「课时」「学习分析」作为教师课程详情入口。

## 页面规则

- 课程管理：展示课程名称、资料数、题目数、创建时间；提供进入课程详情、编辑、删除。
- 课程详情（`/teacher/courses/:courseId`）：只保留**课程信息 + 阶段与资料管理**；可维护阶段、上传/编辑/删除资料、预览与重建预览；**无**课时 CRUD、课时排序、富文本编辑器、学习分析。
- 班级管理：班级按课程创建和筛选；查看学生只读；学生增删进入学生管理页。
- 发布作业：选择题目并发布为作业，不提供公告正文；目标班级支持多选且来自同一课程。
- 学生成绩：承接 `/teacher/grades?tab=tasks&task_id=...`，直接展示作业完成情况。
- 作业完成：从发布作业页跳转，展示指定作业的完成/未完成学生列表及超时情况。
- 资料管理：若侧栏仍保留入口，应以课程详情中的阶段资料管理为主路径；上传和筛选按课程，不再按章节；教师可删除自己课程下的资料，公共课程同步到教师课程副本里的资料也可删除，但不会影响公共课程源内容。
- 教师课程详情中的资料必须展示图文信息，包括封面、摘要、类型、大小、页数或时长、预览状态；不能只展示标题。
- 教师端资料预览与学生端共用站内预览组件，教师可检查 PDF / 视频实际打开效果。
- 题库管理：题库为全站共享，教师可按题型和标签筛选；教师可新增和编辑题目，不提供删除入口，删除只由管理员端处理。

## 学生端课程学习（资料唯一主路径）

- `/learn`：公开教程卡片，只展示资料数量，不展示课时数或学习百分比，不请求课时/进度接口。
- `/learn/course/:courseId`：仅阶段资料目录 + `MaterialInlineReader` 直读 + 资料导读；无课时 Tab、完成本课、上一课/下一课、进度心跳。
- 旧链接 `?lesson_id=` / `?tab=lessons` 由路由自然忽略，直接进入资料阅读，不白屏。
- 无资料时显示中文空状态；不出现“登录后可保存学习进度”等无法兑现文案。

## 交互底线

- 危险操作必须确认。
- 后端拒绝删除课程或班级时，页面必须给出可理解提示。
- 空列表需要有中文空状态。
- 所有页面文案使用正常中文，禁止乱码和英文占位文案。
