# 教师端前端验收要点

## 导航与命名

教师端主导航固定为：概述、管理课程、班级管理、发布作业、学生成绩、作业完成、作品审核、学生管理、题库管理、AI 点名。

- 不再出现“任务发布”“学生数据”作为教师端入口名。
- 不再保留独立「资料管理」页面、`/teacher/materials` 路由或导航入口；旧地址命中全局 404，不兼容跳转。
- 不再出现章节相关入口、筛选项或详情页。
- 不再出现「课时」「学习分析」作为教师课程详情入口。

## 页面规则

- 管理课程（列表）：展示课程名称、资料数、题目数、创建时间；已添加课程提供进入详情、编辑、删除。列表路径仍为 `/teacher/courses`，可从详情页「返回课程管理」进入。
- 管理课程（详情）：教师维护课程资料的唯一入口，只保留**课程信息 + 阶段与资料管理**；资料区继续覆盖上传、编辑、删除、站内预览和预览重建，**无**课时 CRUD、课时排序、富文本编辑器或学习分析。左侧导航「管理课程」与列表操作「管理课程」指向同一详情页（最近管理的课程，或首门已添加课程）；无课程时回落列表。不提供跨课程资料总表、关键词搜索或服务端分页。
- 班级管理：班级按课程创建和筛选；查看学生只读；学生增删进入学生管理页。
- 发布作业：选择题目并发布为作业，不提供公告正文；目标班级支持多选且来自同一课程。
- 学生成绩：承接 `/teacher/grades?tab=tasks&task_id=...`，直接展示作业完成情况。
- 作业完成：从发布作业页跳转，展示指定作业的完成/未完成学生列表及超时情况。
- 课程详情中的资料必须展示图文信息，包括封面、摘要、类型、大小、页数或时长、预览状态；不能只展示标题。
- 管理课程详情「阶段资料」使用扁平紧凑行式展示（封面 + 多行摘要 + 元信息 + 操作）：摘要约 3 行可见、超长再截断，悬停可看全文；避免回到整段预览正文撑高列表。
- 教师端资料预览与学生端共用站内预览组件，教师可检查 PDF / 视频实际打开效果。
- 题库管理：题库为全站共享，教师可按题型和标签筛选；教师可新增和编辑题目，不提供删除入口，删除只由管理员端处理。

## 学生端课程学习（资料唯一主路径）

- `/learn`：公开教程卡片，只展示资料数量，不展示课时数或学习百分比，不请求课时/进度接口。
- `/learn/course/:courseId`：仅阶段资料目录 + `MaterialInlineReader` 直读 + 资料导读；无课时 Tab、完成本课、上一课/下一课、进度心跳。
- 旧链接 `?lesson_id=` / `?tab=lessons` 由路由自然忽略，直接进入资料阅读，不白屏。
- 无资料时显示中文空状态；不出现“登录后可保存学习进度”等无法兑现文案。

## 交互底线

- 危险操作必须确认。
- 后端拒绝删除课程或班级时，页面必须给出可理解提示。
- 空列表需要有中文空状态。
- 所有页面文案使用正常中文，禁止乱码和英文占位文案。
