# 单机 1GiB 视频上传修复

> **状态：已实施，待源代码受控发布与浏览器大文件实测**
>
> **日期：2026-07-18**

## 目标

在不增加服务器资源和不引入对象存储的前提下，使教师与管理员能够上传不超过 1GiB 的课程视频，修复前端 10 秒超时和 Nginx 请求缓冲导致的上传中断。

## 范围

- 上传接口增加视频预估大小和本地磁盘安全余量校验，单 worker 下同一时间只允许一个视频上传。
- 教师课程详情、管理员公共课程的选文件校验、上传进度和可理解的失败提示。
- Nginx 精确 `/api/upload` 路由关闭请求缓冲并放宽上传相关超时；仓库样例和服务器生效配置同步。
- 不改数据库结构，不支持断点续传，不提高实际视频文件的 1GiB 上限。

## 验收

- 140MB 和接近 1GiB 的 MP4 可在浏览器完成上传并创建资料。
- 超过 1GiB、磁盘余量不足、并发视频上传均返回中文业务提示。
- 普通 API 继续保持 10 秒 Axios 超时；只有视频上传使用 60 分钟超时。
- Nginx 配置通过 `nginx -t` 后 reload，独立日志可定位上传问题。

## 服务器部署影响

本轮已于服务器备份 `/etc/nginx/conf.d/tongshi.conf` 后完成 `nginx -t` 与 reload，精确上传路由和独立日志已生效。源代码仍需按提交锁定流程拉取、重启 `tongshi-backend.service`、重建并部署前端静态资源；无需数据库迁移、环境变量调整或服务器扩容。发布后仍需在浏览器实测 140MB 和接近 1GiB 视频。
