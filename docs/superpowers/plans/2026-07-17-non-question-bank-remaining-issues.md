## 二、P0：课程阶段 / 资料删除（明确要修）

### 2.1 用户现象

| # | 现象 | 出现位置 |
|---|------|----------|
| 1 | 删除 PDF/资料后，数据库已删（软删），页面卡片仍在 | 教师「管理课程」详情；管理「公共课程」也可能受同类逻辑影响 |
| 2 | 因页面/关系里仍像「有资料」，阶段删不掉 | 同上 |
| 3 | 期望：删阶段时弹确认 → 确定后**阶段 + 其下资料一起删** | 教师端 + 管理端规则一致 |

### 2.2 根因（已核对代码）

| 现象 | 根因 |
|------|------|
| 删资料后页面仍显示 | 资料走 **软删除**（`deleted_at`）。教师课程详情 `build_course_detail`（`course_response_service.py`）用 `stage.materials` / `course.materials` 组装列表时，**未过滤** `deleted_at is None`，软删行仍返回前端。 |
| 删阶段仍被拦 | 后端 `delete_stage`（`course_stage_service.py`）用 `if stage.materials:` 判断，关系含软删行也会拦截；前端 `stage.materials.length > 0` 在幽灵资料存在时同样拦截。 |
| 交互不符合预期 | 产品要「确认后级联删」；当前是「有资料只 warning、禁止删阶段」。 |

补充：

- 管理端 `list_public_materials` **已有** `deleted_at` 过滤，单份资料列表相对干净。  
- 删阶段两端都走 `delete_stage` + 前端拦截，**级联与「只认活跃资料」**仍要在服务层统一修。  
- 前端 `handleDeleteMaterial` / `removeMaterial` 代码里会 `loadCourse()` / `fetchContent()`，**不是单纯忘了调刷新**；主因是读路径带回幽灵数据。

### 2.3 目标行为

1. 软删资料后，课程详情 / 阶段下列表 **立即不再出现** 该资料（刷新后也一致）。  
2. 删除阶段：弹出确认（说明会删除阶段下资料）→ 确认后：  
   - 阶段下**活跃**资料软删除；  
   - 再删除阶段本身（阶段表当前无软删字段，保持现有物理删阶段行）。  
3. 教师课程详情与管理员公共课程：**同一套删除规则**；管理端同步教师副本的范围差异保留。  
4. 兼容：接口默认不传级联时，有活跃资料仍可拒绝（避免误调旧客户端）；前端正式入口固定传级联。

### 2.4 建议改动点（实施时按此动手）

| 层 | 文件 | 改什么 |
|----|------|--------|
| 读路径 | `backend/app/services/course_response_service.py` | `build_course_detail` 中阶段资料、未分类资料只保留 `deleted_at is None` |
| 写路径 | `backend/app/services/course_stage_service.py` | `delete_stage(..., cascade_materials=False)`：活跃资料判断；级联时对活跃资料 `soft_delete` 再 `db.delete(stage)` |
| 教师 API | `backend/app/api/v1/routes/stage_routes.py` | 查询参数 `cascade_materials`，传入服务层 |
| 管理 API | `backend/app/api/v1/routes/admin_public_course_routes.py` | 删公共阶段同样支持级联；处理副本阶段时只认活跃资料 |
| 前端 API | `frontend/src/api/course.ts`、`adminPublicCourse.ts` | 删除阶段支持 `cascadeMaterials` |
| 教师页 | `frontend/src/views/teacher/TeacherCourseDetail.vue` | 去掉「有资料禁止删」；确认文案说明会删资料；请求级联 |
| 管理页 | `frontend/src/views/admin/AdminPublicCourses.vue` | 同上 |
| 测试 | 新建后端用例 + 前端静态断言 | 软删后详情无幽灵；级联后阶段不存在、资料 `deleted_at` 有值 |
| 文档 | `backend/docs/项目修改记录.md`、`AGENTS.md` | 记录行为约定与部署影响 |

### 2.5 不做（本问题线）

- 不改共享题库页面与入库规则  
- 不给阶段表新增 `deleted_at`（不扩表）  
- 不改资料文件物理清理 / 回收站策略（仍走既有 soft_delete）  
- 不统一教师/管理整页皮肤  

### 2.6 验收

- 删 PDF 后列表卡片消失，资料数减少  
- 阶段下有资料时点删除 → 确认 → 阶段与资料均不可见  
- 教师端、管理公共课程行为一致（同步语义可不同）  
- 相关 pytest / 静态测试 / `npm run type-check` / `npm run build` 通过  

### 2.7 服务器部署影响（实施并发布后）

需要：拉代码、重启后端、重新构建并发布前端。  
不需要：数据库迁移、环境变量、Nginx 调整。  
**仅本文档存在、未改代码时：不需要服务器修改。**

## 三、非题库：管 / 教职责边界（对照摘要）

以下与共享题库展示无关，实施阶段/资料修复时不必改这些页面，但治理上应心里有数。

| 域 | 教师端 | 管理员端 | 是否要对齐产品 |
|----|--------|----------|----------------|
| 自有课 / 公共课副本 | 列表、详情、阶段、资料、加公共课 | 公共课**源**维护，同步到副本 | 规则一致（删阶段级联等），对象不同（副本 vs 源） |
| 班级 / 作业 / 成绩 / 作业完成 / 点名 | 有 | 无 | 教师独有，不对齐 |
| 学生作品审核 | 有 | 无 | 教师独有 |
| 密码重置 | 学生管理内、**本班** | 独立菜单、**全站** | 能力同、范围不同，可保持 |
| 教师账号 | 无 | 创建/导入/重置/删除 | 管理独有 |
| 公益/读书会图文 | 无 | 内容管理 | 管理独有（勿与作品审核混淆） |
| 回收站 / 审计日志 | 无 | 有 | 管理独有 |

**原则（建议日后写入 `AGENTS.md`，本轮可不改代码）：**

1. 课程阶段/资料删除：确认后可级联软删阶段下活跃资料；读路径必须过滤软删资料。  
2. 教师课程详情与管理员公共课程在删除规则上保持一致；同步范围可以不同。  
3. 班级教务、作业、成绩、作品审核属教师；账号、公共课源、回收站、审计、图文属管理员。  
4. 密码重置：教师本班、管理全站。
